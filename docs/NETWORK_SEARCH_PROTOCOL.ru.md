# Протокол Поиска в Сети (RU)

Сетевой поиск обязателен, если:
1. факт нестабилен (API, версии, правила, цены, политики),
2. пользователь просит проверить актуальную информацию,
3. локальной документации недостаточно,
4. риск ошибки высокий.

## Приоритет Источников
1. Официальная документация вендора/проекта (primary).
2. Официальные стандарты/спецификации.
3. Локальные контракты и docs репозитория.
4. Вторичные источники (статьи/блоги) только как дополнение.

## Обязательные Шаги
1. Сформулировать точный вопрос одной строкой.
2. Выполнить 2-4 целевых запроса.
3. Открыть первоисточники и подтвердить ключевые факты.
4. Сверить вывод с локальными контрактами.
5. Зафиксировать evidence с URL и timestamp.

## Шаблон Evidence-Блока
```md
### Network Cross-Check
- Question: <что проверяем>
- Sources:
  - <url_1>
  - <url_2>
- Checked at (UTC): <timestamp>
- Decision: <принятое решение>
- Repo impact: <какие файлы/контракты обновлены>
```
