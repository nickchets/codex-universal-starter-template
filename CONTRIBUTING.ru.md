# Руководство По Вкладу

Спасибо за вклад.

## 1. Сначала Контракты
Перед реализацией синхронизируйте:
- `PROJECT_MANIFEST.md`
- `ACTION_PLAN.md`
- `docs/DIRECTIVE_REGISTER.md`

Инварианты доски должны оставаться валидными:
- `DOING == 1`
- `NEXT <= 3`

## 2. Цикл Изменений
1. Берем текущую карточку `DOING`.
2. Делаем минимальный безопасный дифф.
3. Прогоняем верификацию.
4. Фиксируем evidence.
5. Делаем корректный переход состояния доски.

## 3. Верификация
Минимальный baseline:
```bash
./tools/verify_fail_fast.sh
python3 tools/dev_harness_server.py workflow-context-gate --label contrib_check --strict-medium
```

## 4. Качество Коммитов
- Коммиты должны быть фокусными и обратимыми.
- При необходимости указывайте verify/evidence-контекст в сообщении коммита.
- Не сливайте изменения с проваленной проверкой.

## 5. Чеклист Для PR
- [ ] Scope соответствует карточке `DOING`.
- [ ] Тесты/проверки проходят.
- [ ] `EVIDENCE_LOG.md` обновлен.
- [ ] Переход по доске валиден.
- [ ] Нет конфликтующих изменений в governance-доках.
